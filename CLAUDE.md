# 串口
- 程序启动自动连接串口COM7，波特率57000

# udp通信
- 本app监听6667端口，对外发送信息的端口6666
- 本机需要给局域网中的其他设备发送和接受信息
- 到达目标点位后，发送对应点位的名称，比如，1号点发"P1", 2好点发“P2”，依此类推
- udp收到1，移动到1号点位，收到2，移动到2号点位，依此类推

# 回零
- 程序驱动后，自动开始回零，默认回零方式CCW，方式为2；
- 回零完成后，自动去第一个点位

# 单位换算
- 1mm=250脉冲、
- 界面上的位置数据单位统一为mm

# config.json
- 电机的参数比如速度，加速度，减速带，回零速度，回零offset，点位列表等数据暴露在config.json文件里
- 点位判断：实际位置与目标位置相距distTolerance=10mm时，判定为到达目标位置。


# 按钮
- 下一个： 去下一个点位
- 上一个： 去上一个点位
- 起点：去start点位。  start点位在config.json文件里配置，可能是第一个点，也可能是最后一个点，或者任意一个点
- CCW回零：以CCW方向回零
- CW回零：CW方向回零
- 停止：电机停止运动
- 自动巡航：电机挨个移动到点位列表里的所有点位，周而复始
- 添加点位：添加当前位置到点位列表
- Go：移动到当前点位对应的目标位置
- 编辑：点位列表中激活的点位数值变为可输入状态，通过下方的-100，-10，-1，+1，+10，+100按钮编辑，
- -100，-10，-1，+1 +10，+100， 当前编辑状态的数值加或者减对应数值
- 删除：删除当前激活的点位，重新安排点位的序列号
- 保存：存储当前点位位置数值到config.json文件，并退出编辑状态。
- 启动：电机移动到输入框对应的位置。
- CCW100：以当前位置为基准，CCW方向移动100mm
- CW100：以当前位置为基准，CW方向移动100mm
- 清除全部点位：位置列表清零，删除全部点位。该按钮要弹出确认界面，防止误操作。
- 设为起点：把列表中当前激活的点位设置为起点，对应config中的startPoint
- 点位列表里的点位，可以被选中，选中之后可以进行编辑，编辑位置数据。
- 点击ccw回零和cw回零按钮时，清除之前的状态和命令，执行当前按钮的动作
- 点位添加，修改，删除的同时，保存/更新config.json文件里的点位数据，启动程序时，从该文件加载之前的点位数据

# 状态标签
- 回零完成：已完成/未完成/回零进行中
- 串口：已连接/未连接
- UDP-Msg：显示接收到的udp信息
- 当前点位：当前位置对应点位列表中的那一个
- ui界面中的点位列表，哪个点位是起点，即config.json 中的 startPoint，放置一个红色圆点作为标记


# 开关(Toogle)

- 自动初始化：on=程序启动后，自动开始进行回零等初始化动作，off=程序启动不自动回零，回零状态为已完成回零
- 发送位置：on=通过udp发送实时位置数据，off=不发送位置数据。电机在移动过程中，通过upd发送实时位置数据，每秒30次，目标端口6666，移动时才发送，到达目标位置后，停止发送。
- 开关状态暴露在config.json文件里，开关状态有变动，自动保存，下次启动从config文件里加载对应状态
